# 实验报告自评辅助系统 需求文档 (Requirements)

## 1. 引言
本文档依据《实验报告自评辅助系统需求规格说明书 (SPEC)》展开，用用户故事和验收标准形式详细描述系统各功能模块的行为，作为开发、测试和验收的依据。

## 2. 用户故事与功能需求

### US-01：学生登录与个人中心
- **作为** 学生
- **我想要** 使用学号和密码安全登录系统
- **以便** 只有本人可以查看和管理自己的实验报告与自评记录

**验收标准：**
1. 登录页面需包含学号、密码输入框和登录按钮。
2. 系统验证学号密码正确后，生成 JWT 令牌，并重定向至个人工作台。
3. 若连续 5 次输入错误密码，账号锁定 15 分钟。
4. 学生可在个人中心查看自己的姓名、班级、头像（可选）及历史自评次数。
5. 预留教师登录入口（灰色不可用状态，标注“教师端即将开放”）。

### US-02：实验报告上传
- **作为** 学生
- **我想要** 上传 `.docx` 或 `.pdf` 格式的实验报告文件
- **以便** 系统能够自动解析并评分

**验收标准：**
1. 上传区域支持拖拽和点击选择文件，限制文件类型为 `.docx` 和 `.pdf`，单文件大小 ≤ 20MB。
2. 上传后显示进度条，完成后文件出现在“我的报告”列表中，状态为“待解析”。
3. 禁止上传手写扫描件（若 PDF 为纯图片且无可提取文本，系统提示“请上传电子文档，暂不支持手写扫描件”）。
4. 上传成功后系统异步进行解析，前端每 2 秒轮询解析状态，完成后自动跳转至评分结果页。

### US-03：报告自动解析与结构识别
- **作为** 学生
- **我想要** 系统自动识别我报告中的封面信息和各个实验模块
- **以便** 我能快速看到报告结构是否缺失

**验收标准：**
1. 系统正确提取封面中的字段：学校名称、课程名称、指导教师、学院、年级专业、班级、学号、姓名。
2. 若某字段仍为原始下划线占位符（如“_________”），标记为“未填写”。
3. 识别实验模块数量（实验1、实验2…），并列出每个模块下的五个子区块：目的、内容、过程、结果、总结。
4. 每个区块标注内容状态：“已填写”或“未填写”（空或仅保留占位符）。
5. 解析结果以结构化卡片形式展示，未填写区块红色高亮。

### US-04：默认评分量规查看
- **作为** 学生
- **我想要** 在评分前了解评分维度和权重
- **以便** 知道报告哪些方面会被重点关注

**验收标准：**
1. 在评分结果页顶部可展开“评分标准”面板。
2. 以表格形式清晰展示五大维度、权重及简要说明。
3. 该标准由系统内置，当前版本不支持学生自定义。

### US-05：自动评分与结果展示
- **作为** 学生
- **我想要** 系统立即给出各维度的得分和总分
- **以便** 快速了解报告的质量

**验收标准：**
1. 根据解析结果和本地规则引擎，系统在 5 秒内完成打分（不启用AI时）。
2. 评分结果包含：雷达图（五维）、每个维度的得分/满分、总分。
3. 每个维度可点击查看具体扣分原因（如“实验2 过程区块无数据截图，扣3分”）。
4. 若管理员后台开启了AI评测，且本报告允许使用AI，则总分旁标注“AI辅助评分”，并提供AI评测的简要依据。
5. 学生可以手动拖动滑块微调各维度分数（教师预设允许浮动范围，默认 ±5%），调整后总分自动更新。

### US-06：评语与改进措施生成
- **作为** 学生
- **我想要** 在得分之外获得总结性评语和具体改进建议
- **以便** 知道我该修改哪些地方

**验收标准：**
1. 评语区域展示一段自然语言总结，概括主要优点和不足。
2. 改进措施以编号列表展示，每条均包含：
   - 针对的报告位置（如“实验2 总结”）
   - 问题描述（如“内容为空”）
   - 修改建议（提供模板或示例，如“请补充本次实验的收获，例如：‘通过本次实验我掌握了……’”）
3. 当改进措施对应明确区块时，前端高亮其卡片边框。
4. 每条改进措施旁有“已修改”勾选框（仅UI标记帮学生追踪），不影响系统记录。

### US-07：自评工作流：修改重传与确认
- **作为** 学生
- **我想要** 根据改进建议修改报告后重新上传，看到分数变化；确认自评后锁定结果
- **以便** 完成最终规范的自评过程

**验收标准：**
1. 首次上传评分后，页面上提供两个动作按钮：“重新上传修改版”和“确认并提交自评”。
2. 选择“重新上传修改版”：
   - 允许上传新文件，上传成功后视为新版本。
   - 新版本解析评分后，与历史版本并列展示，提供版本间分数对比（差异值用绿色上升箭头、红色下降箭头）。
3. 历史版本至少保留最近 5 版，支持查看各版详细评分和评语。
4. 选择“确认并提交自评”：
   - 弹出最终确认对话框，显示最终分数（系统分或学生微调后的分数）、评语清单。
   - 点击确认后，该报告版本状态变为“已确认”，禁止再修改或重新上传。
   - 生成“自评报告卡片”，可下载为 PDF（包含得分图表、评语、改进措施记录）。
5. 未确认的报告可随时返回编辑，状态为“草稿”。

### US-08：自评历史与数据可视化
- **作为** 学生
- **我想要** 查看我过去所有报告的自评记录和分数变化趋势
- **以便** 追踪我的实验写作水平进步情况

**验收标准：**
1. “我的历史”页面以时间轴形式展示所有已确认的自评记录。
2. 每条记录可展开查看当时的雷达图和评语。
3. 提供“分数趋势”折线图，横轴为报告提交顺序，纵轴为总分。
4. 支持按学期或课程名称筛选。

### US-09：教师端班级查看与导出（预留）
- **作为** 教师（预留角色）
- **我想要** 查看所负责班级学生的自评完成情况和统计数据
- **以便** 了解整体学习效果

**验收标准（本期V1.0仅实现基础API，前端界面可后补）：**
1. 提供 API 返回指定班级的学生列表及各自最近一次自评的总分、提交状态。
2. 提供 API 导出班级成绩为 Excel，包含各维度得分和评语摘要。
3. 前端简单表格展示（若开发周期允许，可用管理后台模板完成）。

### US-10：系统管理员配置
- **作为** 管理员
- **我想要** 管理系统用户、评分规则和AI接口
- **以便** 维护系统的正常运行

**验收标准：**
1. 管理员可后台手动添加/导入学生和教师账号。
2. 提供评分量规编辑界面：可修改五大维度的权重、各子检测项的扣分规则。
3. 可配置是否启用AI评测、AI模型选择及API Key。
4. 可设置学生自评分数的允许手动调整范围（百分比）。
5. 支持查看系统日志和生成操作统计报表。

## 3. 非功能需求验收

| 需求点 | 验收方法 |
|--------|----------|
| 10MB报告15秒内解析评分 | 使用规定大小文档测试，要求95%请求在此时间内完成 |
| 同时在线200人 | JMeter压测，200并发登录并上传，系统响应时间不超2秒 |
| 文件加密存储 | 检查服务器存储目录，文件内容非明文 |
| AI云端评测开关 | 管理员关闭后，学生端评分不显示AI相关标注 |
| 浏览器兼容 | 在Chrome 90+、Edge 90+、Firefox 88+上全流程通过测试 |
| 容器化部署 | 提供单命令 `docker-compose up` 启动全部服务 |

## 4. 界面原型要点（非视觉稿）
- **登录页**：居中卡片，学校Logo，学号密码，提示注册请联系管理员。
- **工作台**：左侧导航（上传报告、我的历史、评分标准），主区域显示最近报告状态。
- **上传页**：拖拽区+格式限制说明。
- **评分结果页**：顶部雷达图+总分，中部维度得分卡片，下部评语与改进措施列表，底部操作按钮。
- **历史页**：左侧时间轴，右侧详情面板。

## 5. 需求优先级
- P0（MVP）：US-01 至 US-06，即登录、上传、解析、自动评分、评语生成。
- P1（核心流程）：US-07 自评工作流与确认。
- P2（增强体验）：US-08 历史趋势可视化、US-10 管理配置界面。
- P3（扩展）：US-09 教师端完整功能。

## 6. 约束与假设
- **假设**：学生使用的报告模板固定为附录A所示格式（成都东软学院统一模板），模板变动需更新解析规则。
- **假设**：PDF报告为电子生成（非扫描图片），可提取文本；Word报告未加密。
- **约束**：AI评测功能依赖外部大模型服务，需网络连接，且应遵守学校数据隐私规定。
