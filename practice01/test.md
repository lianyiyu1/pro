# 实验报告自评辅助系统 测试文档 (test.md)

## 1. 测试策略
- **单元测试**：覆盖所有评分规则、数据解析、工具函数，保证逻辑正确。使用 `pytest`。
- **集成测试**：覆盖 API 端点与数据库、文件存储交互，验证业务流。使用 `pytest + httpx` 或 Postman/Newman。
- **端到端测试 (E2E)**：模拟真实学生流程，使用 Cypress 或 Playwright。
- **性能测试**：利用 JMeter 进行并发上传与评分压测。
- **安全测试**：手动渗透测试 + 自动化扫描（敏感数据、权限绕过）。

## 2. 测试环境
- **本地开发环境**：Docker Compose 启动 FastAPI + PostgreSQL + Redis。
- **测试数据**：准备标准模板的 `.docx` 和 `.pdf` 文件，包含完整填写、部分缺失、完全空白以及扫描版 PDF 各一份。
- **AI模拟**：AI 评测调用用 mock 服务替代，返回预设评分和评语。

## 6. 性能测试
| 场景 | 目标 | 测试方法 |
|------|------|----------|
| 并发上传 | 200用户同时上传10MB文件，95%响应时间≤15秒 | JMeter线程组200，循环1次 |
| 并发评分调用 | 100用户同时请求评分，解析已完成 | 响应时间≤3秒 |
| 数据库连接池 | 高并发下无连接泄漏 | 持续运行30分钟，验证连接数 |

## 7. 安全测试
| 检查项 | 方法 |
|--------|------|
| SQL注入 | 在登录学号输入框尝试注入语句，应被参数化查询阻止 |
| XSS | 提交含script标签的文件名，前端应转义 |
| 权限提升 | 学生尝试调用教师API，应返回403 |
| 敏感数据泄露 | 检查API响应，不应包含密码哈希、文件绝对路径 |
| 文件上传 | 上传php文件，应被拒绝；上传超大文件触发限制 |
| AI数据发送 | 启用AI时，抓包检查发送内容是否脱敏（如隐藏姓名学号） |
