# 实验报告自评辅助系统 需求规格说明书 (SPEC)

## 1. 项目概述
### 1.1 项目名称
**实验报告自评辅助系统**（Experiment Report Self-Assessment System）

### 1.2 项目目标
构建一个多人在线 Web 平台，支持学生上传 `.docx` / `.pdf` 格式的实验报告，系统基于预定义模板自动进行结构解析、多维度打分、生成评语及改进措施，辅助学生独立完成实验报告的自评与优化。

### 1.3 用户角色
- **学生**：上传报告、查看自动评分/评语/改进措施、多次迭代修改、提交最终自评。
- **教师**（可选扩展角色，本期暂不实现核心 CRUD，但预留接口）：查看班级学生自评结果、管理评分量规。
- **系统管理员**：用户管理、系统配置。

## 2. 功能需求
### FR-01：用户认证与基础管理
- FR-01.1：支持学生通过学号+密码或学校统一认证登录。
- FR-01.2：学生只能查看和操作自己的报告与自评记录。
- FR-01.3：预留教师角色，教师可查看所授班级学生列表及自评统计（本期仅预留 API）。

### FR-02：实验报告上传与解析
- FR-02.1：支持上传 `.docx` 和 `.pdf` 格式（不接受手写扫描件，仅解析电子文本、图片、表格信息）。
- FR-02.2：系统自动解析文档，识别封面（学校名称、课程名称、指导教师、学生信息等）及多个实验模块（实验1, 2, 3…）。
- FR-02.3：基于模板中的区块标题（【实验目的】、【实验内容】、【实验过程】、【实验结果】、【实验总结】）进行语义切分，检测每个区块是否存在，内容是否为空（非仅下划线占位符）。

### FR-03：评分量规管理
- FR-03.1：系统内置一套默认评分维度及权重，教师可在后续版本中自定义。默认维度见附录A。
- FR-03.2：支持扣分/加分项配置，并在评分时自动应用。

### FR-04：自动评分引擎
- FR-04.1：根据规则对以下维度进行打分：
  - 封面及整体格式
  - 实验模块结构完整性
  - 内容逻辑质量（目的-内容-过程-结果-总结的关联性）
  - 创新与独立思考
  - 书写规范
- FR-04.2：对于逻辑质量、创新等主观维度，提供两种实现路径（可通过开关配置）：
  - 路径A（本地规则）：关键词匹配、句式检测、内容长度等启发式评分。
  - 路径B（AI评测）：调用大语言模型 API 进行语义理解与评判（需管理员开启并配置 API key）。
- FR-04.3：允许系统评分后，学生可手动微调各项分数（浮动范围由教师预设），最终提交“自评总分”。

### FR-05：评语与改进措施生成
- FR-05.1：根据各维度扣分点和缺失项，自动生成总结性评语（模板+动态填充）。
- FR-05.2：生成具体、可操作的改进措施，逐条列出指向明确的修改建议，并可附带简短示例。
- FR-05.3：改进措施与报告结构锚定，高亮对应段落（如技术可实现，在前端同步定位）。

### FR-06：自评工作流
- FR-06.1：上传报告→自动解析与预评分→查看各维度得分+评语+改进措施→学生选择“修改并重新上传”或“确认自评分数”。
- FR-06.2：系统记录每一次上传版本和对应评分结果，支持历史版本对比。
- FR-06.3：最终确认后锁定自评记录，生成自评报告卡片。

### FR-07：数据可视化与导出
- FR-07.1：个人端展示各维度得分雷达图/条形图及历史分数趋势折线图。
- FR-07.2：教师端（预留）可导出班级自评成绩 Excel 报表，含各维度得分及评语摘要。

## 3. 非功能需求
### NFR-01：性能
- 报告解析及评分应在上传后 15 秒内完成（对大小在 10MB 以内的文档）。
- 支持同时在线用户数 ≥ 200。

### NFR-02：安全与隐私
- 学生报告文件敏感，需加密存储（AES-256）。
- 若启用AI云端评测，需向用户明示并支持关闭，默认使用本地规则评分。
- 身份认证集成 OAuth2.0 或 JWT，防止越权访问。

### NFR-03：可用性
- UI 界面简洁，适合非技术学生操作，上传后能清晰展示评分结果和改进建议。
- 支持 Chrome、Edge、Firefox 现代浏览器。

### NFR-04：可维护性
- 评分规则与量规权重应配置化，便于后期调整而无需重新部署。
- 后端解析模块应设计为可插拔的评测器模式，方便新增维度或切换AI引擎。

### NFR-05：部署
- 支持 Docker 容器化部署，可在校内 Linux 服务器一键启动。
- 数据库使用 PostgreSQL，文件存储可使用本地或 MinIO 对象存储。

## 4. 数据模型概要
- **用户表 (users)**：id, role, student_id, name, class, password_hash, ...
- **报告记录表 (reports)**：id, user_id, filename, upload_time, status (draft/confirmed), ...
- **报告版本表 (report_versions)**：id, report_id, version_number, file_path, parsed_data (JSON), created_at
- **评分结果表 (scores)**：id, version_id, dimension_scores (JSON), total_score, comment, improvements (JSON), is_ai_evaluated
- **评分量规表 (rubrics)**：id, name, dimensions (JSON), created_by, is_active

## 5. 技术栈建议
- 前端：Vue 3 + Element Plus / React + Ant Design
- 后端：Python FastAPI / Django REST Framework
- 文档解析：python-docx, PyMuPDF
- 自然语言处理：jieba 分词 + 自建规则，可选集成 OpenAI / 本地 LLM
- 数据库：PostgreSQL
- 缓存：Redis（用于任务队列与限流）
- 部署：Docker + Nginx

## 附录A：默认评分维度与权重（示例）
| 维度 | 权重 | 检测要点 |
|------|------|----------|
| 封面与格式 | 15% | 必填项是否替换占位符、字体样式一致性 |
| 结构完整性 | 20% | 各实验模块标题存在性、内容非空 |
| 逻辑与内容 | 35% | 目的-内容-过程-结果-总结衔接、数据支撑 |
| 创新与思考 | 20% | 改进建议、异常分析、课外延伸 |
| 书写规范 | 10% | 错别字、图表编号、公式编辑器使用 |
