# 文档质量检查清单

## 1. 主题符合度（对照 topic.md）
- [ ] 全文未出现"排除范围"内的内容
- [ ] 每个"关键问题"均有明确回答
- [ ] 没有连续200字以上的偏题段落
- [ ] 摘要和结论互相对应，且紧扣核心主题
- [ ] 专业与文学的结合自然不生硬

## 2. 风格语气一致性（对照 voice.md）
- [ ] 全文无禁词，情感强度符合设定（3档）
- [ ] 人称呼应且前后统一（无"我们"突然变"我"）
- [ ] 没有过度口语化或网络用语
- [ ] 技术术语使用正确，无滥用夸张修饰
- [ ] 专业术语首次出现有适当解释

## 3. 结构与逻辑（对照 structure.md）
- [ ] 必备章节全部存在，顺序正确
- [ ] 章节标题层级不超过3级
- [ ] 每一章开头有引导句，结尾有小结
- [ ] 段落间逻辑清晰，未出现突然跳跃
- [ ] 文学分析与专业思考衔接自然

## 4. 内容完整性
- [ ] 所有声称"详见下文/图示"的引用均已落地
- [ ] 没有"TODO"、"待补充"等占位符
- [ ] 读书心得：书中观点与个人感悟区分明显（可用"书中提到…"、"我认为…"）
- [ ] 专业结合部分有具体实例支撑
- [ ] 时代使命部分有实际行动规划

## 5. 引用与原创性
- [ ] 所有非原创观点、数据均已标注出处
- [ ] 直接引用用引号，间接引用已改写并标注
- [ ] 未出现大段连续雷同于单一来源
- [ ] 《家》的原文引用准确无误

## 6. 细节纠错
- [ ] 拼写与语法（可使用工具初检）
- [ ] 专有名词、人名、书名号使用正确
- [ ] 日期、版本号、页码无误
- [ ] 数字格式统一规范

## 7. 特定类型附加检查
- **读书心得**：书中观点与个人感悟区分明显；有独立思考，不是简单复述。
- **专业结合**：软件工程思维与文学分析的类比恰当；不生搬硬套技术术语。
- **时代使命**：结合祖国发展需求具体；有个人行动规划，不空洞。

## 8. 字数要求
- [ ] 全文超过3000字
- [ ] 各章节篇幅分配合理

## 9. 最终人工通读
- [ ] 从读者视角读一遍，标记出任何"不易懂""有疑问"之处
- [ ] 检查专业结合部分是否自然流畅
- [ ] 确认时代使命部分是否有真情实感
